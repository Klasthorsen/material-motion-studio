# Figma Workflow

## Library Structure

- Create one page for `Motion tokens`.
- Create one page for `Patterns`.
- Create one page for `Component demos`.
- Create one page for `In-product examples`.

## Suggested Motion Tokens

- Duration tokens: `motion.micro`, `motion.enter`, `motion.exit`, `motion.screen`, `motion.hero`
- Easing tokens: `motion.standard`, `motion.enter`, `motion.exit`
- Distances: `motion.xs`, `motion.sm`, `motion.md`

## Suggested Pattern Inventory

- `Card to detail`
- `Tab switch`
- `Filter swap`
- `Bottom sheet open`
- `Chart highlight`
- `Row expand`

## Working Session With Figma Components

1. Select an existing component or frame.
2. Decide what relationship the next state should communicate.
3. Choose the matching motion pattern.
4. Duplicate the frame and rename layers so Smart Animate can interpolate correctly.
5. Save the approved setup as a documented example in the pattern library.

## Review Questions

- What is the focal point?
- What remains stable?
- What relationship is being communicated?
- Is the motion faster than the user's need to read the content?
- Can the same pattern be reused across similar flows?
